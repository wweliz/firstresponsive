Utilizing the Picturefill Polyfill

The HTML picture element is, in its simplest form, a container for other elements. The beauty of picture is that it allows the developer to serve different versions of an image based on media queries and/or support for a particular image format. It is useful when an image source exists in multiple densities, or when a responsive design (or art direction) calls for a different images on some types of screens.

The picture element is composed of two tags: source, for selecting which image to display, and img, for backward compatibility. The source element has three attributes available:

  • srcset: Specifies the URL of the image you want to show; allows for multiple URLs separated by a comma. Each URL can also be paired with a screen resolution or min-width.
  • media: If the media query evaluates to true, the paired image specified in the srcset attribute will be displayed.
  • sizes: Specifies the set of intrinsic sizes (multiple sizes separated by commas) for the sources described in the srcset.

    • Picturefill is a JavaScript polyfill that enables support for the picture element ()and associated features in browsers that do not yet support them, so you can start using them today!

    A polyfill is a piece of code (in this case, JavaScript) or a plugin that provides the technology that you, the developer, expect the browser to provide natively.

    Because the image is set to have width equal to 100% of the main container, it will scale larger or smaller as you resize the browser window. When the browser window hits the specified breakpoint (set by the media query), it will display a different image.
